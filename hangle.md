# 한글 오토마타 세벌식 추가 분석

이 문서는 세벌식 지원을 위해 추가한 한글 오토마타 변경점을 라인 단위로 정리한다. 라인 번호는 현재 작업 트리의 `Sources/DKSTHangul.h`, `Sources/DKSTHangul.m`, `Sources/InputController.m` 기준이다.

## 1. 자판 종류 상태 추가

파일: `Sources/DKSTHangul.h`

- 3행: `DKSTHangulKeyboardLayout` enum을 선언한다. 엔진 내부에서 자판 종류를 숫자 상태로 들고 있게 만든 진입점이다.
- 4행: `DKSTHangulKeyboardLayoutDubeolsik`은 기존 두벌식 동작이다. 값이 0이라 기본 초기화 상태도 두벌식이 된다.
- 5행: `DKSTHangulKeyboardLayoutSebeolsikFinal`은 세벌식 최종 배열을 의미한다.
- 6행: `DKSTHangulKeyboardLayoutSebeolsik390`은 세벌식 390 배열을 의미한다.
- 24행: `keyboardLayout` 프로퍼티를 추가해 `InputController`가 사용자가 선택한 자판을 엔진에 주입할 수 있게 했다.

## 2. 입력 문자에서 자모로 가는 분기

파일: `Sources/DKSTHangul.m`

- 60행: `mapFromChar:`는 ASCII 문자 하나를 유니코드 한글 자모 코드로 바꾸는 함수다.
- 61행: 현재 자판이 세벌식 390이면 390 전용 매핑 함수로 넘긴다.
- 62행: 390 매핑 결과를 즉시 반환하므로 아래 두벌식/최종 매핑은 타지 않는다.
- 64행: 현재 자판이 세벌식 최종이면 최종 전용 매핑 함수로 넘긴다.
- 65행: 최종 매핑 결과를 즉시 반환한다.
- 68-98행: 위 두 조건에 걸리지 않으면 기존 두벌식 매핑을 그대로 사용한다.

## 3. 세벌식 390 매핑

파일: `Sources/DKSTHangul.m`

- 101행: `mapSebeolsik390FromChar:`는 세벌식 390 배열의 키 문자와 한글 자모 또는 한글 상태 전용 ASCII 문자를 연결한다.
- 102행: 입력 문자 `c`를 `switch`로 나눠 처리한다.
- 103행: 소문자는 일반 입력, 대문자는 Shift 입력으로 해석한다는 주석이다.
- 104-108행: 윗글쇠 줄 왼쪽 영역이다. 예를 들어 `q`는 종성 ㅅ, `w`는 종성 ㄹ, `e`는 중성 ㅕ로 매핑된다. Shift+W는 종성 ㅌ이다.
- 110-114행: 홈 줄 왼쪽 영역이다. `a/s`는 종성 ㅇ/ㄴ, `d/f/g`는 중성 ㅣ/ㅏ/ㅡ로 매핑된다. Shift 조합은 390 배열에 있는 종성 ㄷ/ㄶ/ㄺ/ㄲ 등을 반영한다.
- 116-120행: 아랫글쇠 왼쪽 영역이다. `z/x`는 종성 ㅁ/ㄱ, `c/v/b`는 중성 ㅔ/ㅗ/ㅜ로 매핑된다. Shift+X/C/V는 각각 ㅄ/ㄼ/ㅀ이다.
- 122-134행: 오른손 글쇠 영역이다. 일반 입력은 초성 ㄹ/ㄷ/ㅁ/ㅊ/ㅍ/ㄴ/ㅇ/ㄱ/ㅈ/ㅂ/ㅌ/ㅅ/ㅎ으로 매핑하고, Shift 입력은 390 배열에 맞춰 `<`, `7`, `8`, `9`, `>`, `'`, `4`, `5`, `6`, `0`, `1` 같은 ASCII 출력으로 매핑한다.
- 136-145행: 숫자 줄 매핑이다. 390에서는 일반 숫자 키가 종성/중성/초성으로 쓰인다. Shift+2부터 Shift+0까지는 영문 상태와 같은 `@#$%^&*()`를 출력한다.
- 146행: `/` 키를 중성 ㅗ로 매핑한다. 최종/390 배열 모두 한글 모드에서 `/` 자리에 ㅗ가 있다.
- 147행: `,`/`.` 키는 390에서 Shift와 함께 각각 `2`/`3`을 출력하도록 매핑한다. 일반 입력은 쉼표/마침표 자체를 완료 버퍼에 넣을 수 있게 ASCII로 반환한다.
- 148행: 매핑되지 않은 문자는 `0`을 반환한다. 호출부는 이 값을 “한글 자모 아님”으로 본다.

## 4. 세벌식 최종 매핑

파일: `Sources/DKSTHangul.m`

- 151행: `mapSebeolsikFinalFromChar:`는 세벌식 최종 배열의 키 문자와 한글 자모를 연결한다.
- 152행: 입력 문자 `c`를 `switch`로 나눠 처리한다.
- 153행: 소문자/대문자 처리 규칙이 390과 같다는 주석이다.
- 154-160행: 윗글쇠 줄 왼쪽 영역이다. 최종 배열은 `Q/W/E/R/T` 쪽 Shift 종성이 390과 다르므로 별도 함수로 분리했다. `Shift+R/T`는 각각 종성 ㅀ/ㄽ으로 처리된다.
- 160-164행: 홈 줄 왼쪽 영역이다. 최종 배열의 Shift 조합인 종성 ㄷ/ㄶ/ㄼ/ㄻ, 중성 ㅒ 등을 반영한다.
- 166-170행: 아랫글쇠 왼쪽 영역이다. 종성 ㅁ/ㄱ/ㅋ/ㄳ과 중성 ㅔ/ㅗ/ㅜ를 처리한다.
- 172행: `b`는 중성 ㅜ, `Shift+b`는 배열표에 맞춰 `?`를 출력한다.
- 174-184행: 오른손 글쇠 영역이다. 일반 입력은 초성 ㄹ/ㄷ/ㅁ/ㅊ/ㅍ/ㄴ/ㅇ/ㄱ/ㅈ/ㅂ/ㅌ으로 매핑하고, Shift 입력은 최종 배열의 숫자 출력인 `5/6/7/8/9/0/1/2/3/4`와 작은따옴표 키의 `·`로 매핑한다.
- 185-186행: `n/m`은 초성 ㅅ/ㅎ이고, `Shift+n/m`은 각각 `-`와 `"`를 출력한다.
- 188-192행: 숫자 줄 `1`부터 `5`까지는 종성/중성으로 쓰이고, Shift 조합은 최종 배열의 종성 ㄲ/ㄺ/ㅈ/ㄿ/ㄾ을 반환한다.
- 193-197행: 숫자 줄 `6`부터 `0`까지는 중성 ㅑ/ㅖ/ㅢ/ㅜ와 초성 ㅋ으로 쓰인다. Shift 조합은 영문 상태와 다르게 각각 `=`, `“`, `”`, `'`, `~`를 출력한다.
- 198-202행: `` ` ``, `-`, `=`, `[`, `]`, `\` 키의 최종 전용 기호를 처리한다. 예를 들어 `` ` ``는 `*`, `Shift+``는 `※`, `-`는 `)`, `Shift+-`는 `;`가 된다.
- 198행: `/` 키를 중성 ㅗ로 매핑하고, `Shift+/`는 `!`를 출력한다.
- 199-200행: `,`/`.` 키는 최종 배열에서 Shift 여부와 관계없이 각각 쉼표/마침표로 출력되게 한다.
- 201행: 매핑되지 않은 문자는 `0`을 반환한다.

## 5. 종성 단독 표시 지원

파일: `Sources/DKSTHangul.m`

- 201행: `currentSyllable`은 현재 `_cho`, `_jung`, `_jong` 상태를 화면에 보여줄 문자 하나로 만든다.
- 202행: 초성/중성/종성이 모두 비어 있으면 표시할 문자가 없으므로 `0`을 반환한다.
- 205행: 초성만 있으면 호환 자모로 표시한다.
- 206행: 중성만 있으면 호환 자모로 표시한다.
- 207행: 이번에 추가한 종성만 있는 상태 처리다. 세벌식은 종성 전용 키가 있으므로 종성만 먼저 입력될 수 있다.
- 253-255행: `compatibilityJamo:`가 종성 자모도 표시할 수 있게 했다. 종성 코드는 `compatibilityJamoForJong:`로 넘긴다.
- 259행: `compatibilityJamoForJong:`는 종성 코드를 화면 표시용 호환 자모로 바꾸는 전용 함수다.
- 261-288행: 단일 종성뿐 아니라 ㄳ/ㄵ/ㄶ/ㄺ/ㄼ/ㅄ/ㅀ 같은 복합 종성도 대응 호환 겹자모로 직접 매핑한다. 덕분에 세벌식에서 Shift+S 같은 겹종성 키를 단독으로 눌러도 이상한 종성 코드가 아니라 `ㄶ`처럼 읽을 수 있는 글자로 보인다.

## 6. 세벌식에서 초성 입력 처리

파일: `Sources/DKSTHangul.m`

- 360행: 기존 두벌식 로직은 `초성+중성` 뒤에 자음이 오면 종성 후보로 본다.
- 361행: 세벌식 계열에서는 이 규칙을 끊는다. 세벌식은 종성 전용 키가 따로 있으므로 초성 키가 종성으로 자동 이동하면 안 된다.
- 362행: 지금까지 조합 중인 글자를 완료 버퍼 `_completed`에 넣는다.
- 363행: 새 초성을 `_cho`에 넣고 중성/종성을 비운다. 즉 다음 글자의 초성으로 시작한다.
- 364행: 세벌식 전용 처리가 끝났으므로 `YES`를 반환한다.
- 375행: 이미 `초성+중성+종성`이 있는 상태에서 초성 키가 들어와도 세벌식에서는 새 글자의 초성으로 봐야 한다.
- 376행: 현재 완성 글자를 완료 버퍼에 넣는다.
- 377행: 입력된 초성을 다음 조합의 초성으로 세팅하고 중성/종성을 비운다.
- 378행: 처리 완료를 알린다.

## 6.5. 세벌식 쌍초성 조합

파일: `Sources/DKSTHangul.m`

- 351행: 초성만 조합 중인 상태에서 새 초성이 들어오면 `combineCho:`로 쌍초성 가능성을 먼저 확인한다.
- 352-353행: 두벌식이 아닌 세벌식 계열이고, 조합 가능한 쌍초성이 있으면 이 분기로 들어간다.
- 354행: `_cho`를 ㄲ/ㄸ/ㅃ/ㅆ/ㅉ 같은 쌍초성으로 갱신한다.
- 355행: 쌍초성 조합이 끝났으므로 기존처럼 앞 초성을 확정하지 않고 `YES`를 반환한다.
- 522행: `combineCho:second:`는 초성 두 개를 쌍초성으로 바꾸는 도우미다.
- 523-527행: ㄱ+ㄱ, ㄷ+ㄷ, ㅂ+ㅂ, ㅅ+ㅅ, ㅈ+ㅈ을 각각 ㄲ, ㄸ, ㅃ, ㅆ, ㅉ으로 바꾼다.
- 528행: 그 외 조합은 쌍초성이 아니므로 `0`을 반환한다.

## 7. 세벌식 종성 전용 키 처리

파일: `Sources/DKSTHangul.m`

- 392행: 입력 자모가 종성 범위이면 새 분기로 들어간다. 이 분기가 세벌식 지원의 핵심이다.
- 393행: 현재 초성과 중성이 모두 있으면 완성 중인 음절에 종성을 붙일 수 있다.
- 394-395행: 아직 종성이 없으면 입력 종성을 그대로 `_jong`에 넣는다.
- 396행: 이미 종성이 있는 상태라면 겹받침 가능성을 본다.
- 397행: 기존 `combineJong:` 함수를 재사용해 현재 종성과 새 종성이 겹받침을 만들 수 있는지 계산한다.
- 398-399행: 겹받침이 가능하면 `_jong`을 겹받침 코드로 갱신한다.
- 400-402행: 겹받침이 불가능하면 현재 음절을 완료하고, 새 종성을 단독 종성 상태로 시작한다.
- 405행: 초성+중성 조합이 없는 상태에서 종성 키가 들어온 경우다.
- 406-407행: 이미 조합 중인 자모가 있으면 먼저 완료 버퍼로 보낸다.
- 409행: 입력 종성을 단독 종성 상태로 저장한다. `currentSyllable`의 207행 덕분에 화면에는 호환 자모로 보인다.

## 9. 숫자와 기호 키 입력 문자화

## 10. 입력 컨트롤러와 엔진 연결

파일: `Sources/InputController.m`

- 92행: 사용자 설정이 아직 없을 때 기본 자판을 두벌식으로 등록한다.
- 741행: `isHangulKeyCode:`는 어떤 keyCode를 IME가 한글 입력으로 가로챌지 판단한다.
- 743-769행: 알파벳 영역은 두벌식과 세벌식 모두에서 한글 입력에 쓰이므로 항상 `YES`를 반환한다.
- 770-784행: 숫자와 일부 기호 키는 세벌식에서만 한글 자모 또는 세벌식 전용 ASCII 출력으로 쓰인다. 여기에는 `,`, `/`, `.` 키도 포함된다.
- 785행: 현재 자판이 두벌식이 아닐 때만 숫자/기호 키를 한글 입력으로 가로챈다. 두벌식 사용 중에는 숫자와 문장부호가 정상적으로 원래 문자로 입력된다.
- 1239-1241행: 사용자 정의 Shift 매핑은 두벌식에서만 동작하게 제한한다. 세벌식에서는 Shift 자체가 배열의 일부라 이 기능이 먼저 가로채면 390 숫자열 같은 기본 동작이 깨진다.
- 990-991행: 저장된 `DKSTHangulKeyboardLayout` 설정값을 읽는다.
- 992-993행: 저장값이 `sebeolsik`이면 엔진 상태를 `DKSTHangulKeyboardLayoutSebeolsikFinal`로 둔다. 예전 “세벌식” 저장값과 호환된다.
- 994-996행: 저장값이 `sebeolsik-390`이면 엔진 상태를 390으로 둔다.
- 997-998행: 그 외 값이거나 설정이 없으면 두벌식으로 둔다.
- 1000행: 결정된 자판 상태를 `DKSTHangul` 엔진의 `keyboardLayout` 프로퍼티에 전달한다.

## 11. 동작 예시

- 두벌식 `r` + `k`: `mapFromChar:`가 두벌식 매핑을 사용해 ㄱ + ㅏ로 처리하고 `가`를 만든다.
- 세벌식 최종 `k` + `f`: `k`는 초성 ㄱ, `f`는 중성 ㅏ로 처리되어 `가`를 만든다.
- 세벌식 최종/390 `k` + `/`: `k`는 초성 ㄱ, `/`는 중성 ㅗ로 처리되어 `고`를 만든다.
- 세벌식 390 `m` + `f` + `s`: `m`은 초성 ㅎ, `f`는 중성 ㅏ, `s`는 종성 ㄴ으로 처리되어 `한`을 만든다.
- 세벌식 390 `k` + `k` + `f`: 초성 ㄱ 두 번이 ㄲ으로 합쳐지고, `f`가 중성 ㅏ가 되어 `까`를 만든다.
- 세벌식 390 `Shift+n/m/,/./j/k/l/u/i/o`: 각각 `0`부터 `9`까지 출력한다.
- 세벌식 390 `Shift+b/g/t/y/p/h`: 각각 `!`, `/`, `;`, `<`, `>`, `'`를 출력한다.
- 세벌식 390 `Shift+2~0`, `Shift+;`, `Shift+'`, `Shift+/`: 영문 상태와 같은 `@#$%^&*():\"?`를 출력한다.
- 세벌식 390 `Shift+w/s/d/x/c/v`: 각각 종성 ㅌ/ㄶ/ㄺ/ㅄ/ㄼ/ㅀ으로 처리된다.
- 세벌식 390에서 `Shift+s/d/x/c/v`를 단독으로 누르면 각각 `ㄶ`, `ㄺ`, `ㅄ`, `ㄼ`, `ㅀ`으로 표시된다.
- 세벌식 최종 `Shift+y/u/i/o/p/h/j/k/l/;`: 각각 `5`, `6`, `7`, `8`, `9`, `0`, `1`, `2`, `3`, `4`를 출력한다.
- 세벌식 최종 `` ` ``/``Shift+` ``: 각각 `*`, `※`를 출력한다.
- 세벌식 최종 `Shift+2/4/5`: 각각 종성 ㄺ/ㄿ/ㄾ으로 처리된다.
- 세벌식 최종 `Shift+6~0`: 영문 상태와 다르게 각각 `=`, `“`, `”`, `'`, `~`를 출력한다.
- 세벌식 최종 `Shift+-`, `Shift+=`, `Shift+[`, `Shift+]`, `Shift+\`: 각각 `;`, `+`, `%`, `/`, `\`를 출력한다.
- 세벌식 최종 `Shift+'`, `Shift+,`, `Shift+.`, `Shift+/`: 각각 `·`, `,`, `.`, `!`를 출력한다.
- 세벌식 최종 `Shift+n/m/b`: 각각 `-`, `"`, `?`를 출력한다.
- 세벌식 최종 `Shift+s/d/x`: 종성에서는 각각 ㄶ/ㄼ/ㅄ으로 조합되고, 단독 입력에서도 `ㄶ`, `ㄼ`, `ㅄ`처럼 호환 자모로 표시된다.
- 세벌식에서 `mfsf`: `한` 뒤의 `f`는 받침 ㄴ을 다음 초성으로 옮기지 않고 독립 중성 ㅏ로 시작한다. 결과는 `한ㅏ` 흐름이다.
